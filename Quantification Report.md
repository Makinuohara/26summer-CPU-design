

量化评估报告：单周期 vs 五级流水线CPU性能对比


一、测试目的

本实验旨在对单周期CPU与五级流水线CPU进行全面的性能量化评估，从时钟频率（Fmax）、CPI（每指令周期数）、吞吐量（MIPS）及硬件资源占用四个维度进行对比分析，验证流水线架构在FPGA实现中的性能优势与设计权衡。


二、测试环境

2.1 硬件平台
- FPGA开发板：Nexys 4 DDR
- FPGA芯片：Xilinx Artix-7 XC7A100TCSG324-1
- 可用资源：LUT 63,400 / FF 126,800 / BRAM 135 / DSP 240

2.2 设计参数
- 项目：单周期CPU / 五级流水线CPU
- 流水线级数：1级 / 5级（IF→ID→EX→MEM→WB）
- 冒险处理：无 / 数据前递 + Load-Use暂停 + 分支冲刷 + 中断排水
- Cache：无 / 直接映射D-Cache（16行×4字）
- 时钟约束：100 MHz

2.3 测试工具
- 时钟频率（Fmax）：Vivado 2018.3 Timing Report
- 资源占用：Vivado 2018.3 Utilization Report
- CPI：Vivado Simulator 

三、测试结果与数据分析

3.1 时钟频率评估（Fmax）

评估方法：
Fmax (MHz) = 1000 / (Clock Period - WNS)
其中时钟约束周期为10.000 ns（对应100 MHz）。

【单周期CPU】
- 约束时钟周期：10.000 ns
- WNS：-28.099 ns
- TNS：-138760.641 ns
- Failing Endpoints：15,827 / 17,504
- Fmax：26.25 MHz

分析：单周期CPU的WNS为负值（-28.099ns），在100MHz约束下时序不满足。关键路径从PC寄存器出发，经组合逻辑到达寄存器堆，路径延迟达37.959ns，其中布线延迟占91.1%，这是单周期架构的固有限制。

【五级流水线CPU】
- 约束时钟周期：10.000 ns
- WNS：5.686 ns
- TNS：0.000 ns
- Failing Endpoints：0
- Fmax：231.80 MHz

分析：流水线CPU的WNS为正（+5.686ns），完全满足100MHz时序约束。通过将关键路径切分为五个流水级，工作频率提升至231.80MHz。

【频率对比】
- 单周期CPU：26.25 MHz
- 流水线CPU：231.80 MHz
- 提升幅度：↑ 783%

3.2 CPI评估

评估方法：
CPI = cycle / instret
测试程序为中断响应程序，包含CSR读写、中断排水与mret返回。

【单周期CPU】
- CPI：1.00

【五级流水线CPU】
- cycle：70
- instret：35
- flush：15
- CPI：70 / 35 = 2.00

分析：流水线CPI为2.00。主要原因包括中断排水（flush=15）、分支跳转冲刷、CSR操作停顿。在无中断场景下CPI可接近理论值1.0。

3.3 吞吐量评估

评估方法：
吞吐量 (MIPS) = Fmax (MHz) / CPI

【结果对比】
- 单周期CPU：频率26.25 MHz，CPI 1.00，吞吐量26.25 MIPS
- 流水线CPU：频率231.80 MHz，CPI 2.00，吞吐量115.90 MIPS

分析：尽管流水线CPI因中断变为2.00，但由于频率提升近8.8倍，吞吐量达115.90 MIPS，是单周期的4.4倍。

3.4 硬件资源占用评估

【LUT】
- 单周期CPU：2,983 (4.70%)
- 流水线CPU：4,723 (7.45%)
- 变化：↑ 58%

【FF】
- 单周期CPU：8,704 (6.86%)
- 流水线CPU：5,067 (4.00%)
- 变化：↓ 42%

【BRAM】
- 单周期CPU：0
- 流水线CPU：5 (3.70%)
- 变化：+5

【DSP】
- 单周期CPU：0
- 流水线CPU：3 (1.25%)
- 变化：+3

分析：LUT增加58%用于流水线控制逻辑、前递单元、冒险检测单元；FF减少42%因组合逻辑被分散到流水寄存器；BRAM增加因存储模块实现。


四、性能瓶颈与优化方向

4.1 当前瓶颈
1. 中断排水机制：响应中断需排空流水线，flush=15，增加中断延迟和CPI
2. 仅支持字访问：不支持字节/半字访问，数据带宽利用率低
3. 单发射架构：无法利用指令级并行
4. 写直达Cache：每次写操作同时写Cache和DMEM，增加访存压力
5. 演示频率过低：当前381Hz主要为肉眼观察演示，未发挥性能潜力

4.2 优化方向
1. 提高CPU频率（减小分频）→ 吞吐量线性提升，时序收敛难度增加
2. 引入分支预测 → 减少flush，降低CPI，LUT增加
3. 增加I-Cache → 减少取指延迟，BRAM增加
4. 双发射架构 → 提升IPC，资源大幅增加
5. 改为写回Cache → 减少写操作延迟，增加脏位管理


五、结论

本实验成功完成了RISC-V RV32I单周期与五级流水线CPU的量化评估对比：

- 时钟频率：流水线CPU达231.80 MHz，相比单周期26.25 MHz提升783%
- 时序状态：单周期严重违规（WNS=-28.099ns），流水线完全满足（WNS=+5.686ns）
- CPI：单周期1.00，流水线中断场景2.00（flush=15次，正常代价）
- 吞吐量：流水线达115.90 MIPS，是单周期26.25 MIPS的4.4倍
- 资源代价：LUT增加58%（流水线控制逻辑），FF减少42%（组合逻辑分散），BRAM增加5个

核心结论：五级流水线架构将最大工作频率提升783%，吞吐量提升341%，时序从违规收敛到完全满足，硬件资源代价远低于性能收益，验证了流水线作为CPU性能优化核心手段的有效性。
