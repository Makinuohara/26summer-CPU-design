存储层次规划说明


1. 总体架构

本系统采用哈佛结构，指令存储器与数据存储器物理分离，各自拥有独立的地址总线和数据总线，从根本上避免指令访存与数据访存的冲突。

存储层次自顶向下分为三层，访问延迟逐层增加，容量逐层增大：

第一层：CPU寄存器堆
第二层：Cache高速缓存
第三层：主存（RAM）

寄存器堆位于CPU内部，提供零延迟访问，但容量最小。
Cache位于CPU与主存之间，用于缓存近期访问过的数据。
主存由两个独立的RAM模块组成，分别存储指令和数据。


2. 各层次详细说明

2.1 寄存器堆

位置：CPU内部
容量：32个通用寄存器，每个32位
访问延迟：0周期（组合逻辑读出）
作用：提供操作数缓存，减少对下级存储的访问频率

2.2 Cache高速缓存

位置：CPU与主存之间
结构：直接映射
行数：8行
每行容量：1个字（32位）
总容量：8乘以4字节等于32字节
替换策略：直接映射（无替换决策，固定索引）
写策略：写直通加写分配

地址分解：
Cache访问时，32位物理地址分解为三个字段：
Tag字段：位宽28位，对应地址位[31:4]，用于与Cache行中存储的Tag比较，判断是否命中
Index字段：位宽3位，对应地址位[3:2]，用于索引到8行中的某一行
Offset字段：位宽2位，对应地址位[1:0]，用于字内字节偏移，当前按字寻址，暂不使用

2.3 主存RAM

实现方式：两个独立的RAM模块，imem用于指令存储，dmem用于数据存储
底层模块：统一使用ram.v
地址宽度：12位，支持4096个32位字
总容量：2乘以4K乘以4字节等于32KB，指令和数据各16KB
访问延迟：ready信号恒为1，当前版本为单周期访问，后续可扩展为多周期


3. 数据访问流程

3.1 读操作

第一步：CPU发出读地址和读使能。
第二步：Cache并行计算Index和Tag，读取对应行的Tag和Valid位。
第三步：若Valid为1且Tag匹配，则命中，直接返回Cache中的数据。
第四步：若缺失，则从主存读取目标数据，同时将数据填入Cache并更新Tag和Valid。

3.2 写操作

第一步：CPU发出写地址、写数据和写使能。
第二步：若命中，则同时写入Cache和主存（写直通策略）。
第三步：若缺失，则将数据写入主存，同时按写分配策略将数据加载到Cache。


4. 容量与性能预期

寄存器堆：容量32乘32bit，访问延迟0周期
Cache：容量32字节，访问延迟1周期，规律访问时命中率预期百分之九十以上
主存：容量32KB，访问延迟1周期（当前版本）

对于具有良好时间局部性和空间局部性的程序，如循环顺序访问，Cache命中率可达到百分之九十以上。对于随机跳跃访问，命中率会显著下降，需结合具体访存模式进行分析。


5. 文件清单

ram.v：统一RAM底层模块，参数化地址和数据位宽
imem.v：指令存储器（只读），例化ram，写使能固定为0
dmem.v：数据存储器（读写），直接透传ram接口
cache.v：直接映射Cache，8行乘1字，含hit和miss统计
