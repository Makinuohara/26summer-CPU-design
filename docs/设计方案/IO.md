# IO 接口设计方案

## 目标

本部分负责在 A 同学定义的 SoC 数据总线契约上实现板载 I/O 设备，使 CPU 可以通过 `lw/sw` 指令访问 Nexys 4 DDR 的滑动开关、按钮、LED 和八位七段数码管。

当前实现聚焦课程演示需要：

```text
SW[7:0] -> CPU 读取 -> +1 -> LED[7:0]
BTNC 两次锁存 A/B -> CPU 计算 A+B 或 A-B -> 数码管显示 8 位补码结果
BTNU 在第二次锁存前选择本轮减法
```

PS/2、PLIC 和中断请求线暂不实现，相关 `dmem_irq` 输出恒为 0。

上板顶层中 CPU、数据总线和 I/O 寄存器使用 `CLK100MHZ` 分频后的慢时钟运行，七段数码管扫描逻辑继续使用 100MHz 时钟，避免单周期 CPU 直接跑 100MHz 时出现时序风险。

## 地址映射

所有 I/O 设备均挂载在数据总线译码器后，端口遵循 `architect.md` 的统一设备接口。

| 设备 | 地址范围 | 访问属性 | 当前用途 |
| --- | --- | --- | --- |
| 数据存储器 | `0x0000_0000` - `0x07ff_ffff` | R/W | 普通 `lw/sw` 数据访问 |
| 滑动开关 | `0x8000_0008` - `0x8000_000b` | R | 返回 `{16'b0, SW[15:0]}` |
| LED | `0x8000_000c` - `0x8000_000f` | R/W | 写低 8 位到 `LED[7:0]`，`LED[15:8]` 置 0 |
| 七段数码管 | `0x8000_0010` - `0x8000_002f` | R/W | 演示模式写低 8 位补码值，显示为两位十六进制 |
| 用户按钮 | `0x8000_0030` - `0x8000_0033` | R | 返回 `{27'b0, BTN_C, BTN_U, BTN_R, BTN_D, BTN_L}` |

未命中任何设备的地址返回 `dmem_ack=1` 且 `dmem_fault=1`。写入只读设备（SW、BTN）也返回 fault。

## 设备寄存器

### 滑动开关

```text
地址：0x8000_0008
bit[15:0]  SW[15:0]
bit[31:16] 保留，读 0
```

当前演示程序只使用 `SW[7:0]`。硬件内部对开关输入做两级同步，避免直接把异步板载输入送入 CPU 时钟域。

### LED

```text
地址：0x8000_000c
bit[7:0]   LED[7:0]
bit[15:8]  当前实现固定为 0
bit[31:16] 保留，读 0
```

虽然架构文档保留 16 位 LED 能力，本次交互演示刻意只显示低 8 位，便于和 8 位补码结果保持一致。

### 用户按钮

```text
地址：0x8000_0030
bit[0] BTN_L
bit[1] BTN_D
bit[2] BTN_R
bit[3] BTN_U
bit[4] BTN_C
```

当前演示只使用：

```text
BTN_C：锁存当前 SW[7:0]
BTN_U：在第二次锁存前选择减法
```

按钮输入做两级同步。按键边沿由 CPU 程序通过“处理后等待释放”的轮询方式消除重复触发。

### 七段数码管

地址范围为 `0x8000_0010` - `0x8000_002f`。本实现保留原始段码寄存器空间，同时为演示提供一个简化十六进制显示入口：

```text
0x8000_0010：写入 bit[7:0]，硬件显示为两位十六进制补码
0x8000_0014 - 0x8000_002f：保留为原始段码寄存器
```

例如：

```text
写入 0x03 -> 数码管低两位显示 03
写入 0xff -> 数码管低两位显示 FF，表示 8 位补码 -1
```
