# 系统架构设计

## 概要

本项目基于nexys4ddr开发板实现一个片上系统（SoC），采用哈佛结构，包括RISC-V 32I架构的流水线CPU，内存子系统，IO接口等部分。

目标是支持RISC-V 架构的程序运行。

## 系统结构

### 数据通路

采用哈弗结构,分离指令和数据的通路，两者独立寻址。

#### 控制总线

负责事务语义含，wr线，irq线，busy线等

#### 数据总线

cpu直连，通过数据总线译码器和各外设接口的输出相连

#### 数据地址总线

cpu，总线译码器。

##### 数据总线译码器

板上的硬件资源采用内存映射方式供用户代码访问。

总线译码器承担了设备使能和数据选通两大职责。

设备使能：cpu执行LW/SW指令访存时，由数据总线译码器对地址总线上的地址进行译码，使能对应的设备。

数据选通：根据地址把设备的数据输出选通到数据总线上。

| 设备           | 起始地址    | 结束地址    | 备注                                       |
| -------------- | ----------- | ----------- | ------------------------------------------ |
| 数据存储器     | 0x0000 0000 | 0x07FF FFFF | 路由到数据存储器                           |
| PS/2控制器接口 | 0x8000 0000 | 0x8000 0007 | 两个32位寄存器，用于与USB设备通信          |
| 滑动开关接口   | 0x8000 0008 | 0x8000 000B | 32位字对齐，只用低16位对应开关             |
| LED接口        | 0x8000 000C | 0x8000 000F | 32位对齐，只用低16位                       |
| 数码管接口     | 0x8000 0010 | 0x8000 002F | 路由到数码管控制器，每个寄存器分配32位对齐 |
| 用户按键接口   | 0x8000 0030 | 0x8000 0033 | 5个用户按钮，32位寄存器                    |
| 中断控制器     | 0x8100 0000 | 0x8120 0007 | RISC-V标准大小，实际按需使用               |

#### 指令总线

cpu和指令存储器连到指令总线上。

#### 指令地址总线

cpu给指令存储器发送指令地址。

### 主存

Cache使用BRAM实现，缓存策略自定

#### 数据存储器

存储运行时数据，cpu直接通过该存储器读写数据

#### 指令存储器

存储程序，cpu直接通过该存储器读取指令，不参与数据总线译码，程序通过位流烧录固化。

**DDR的使用**

虽然哈弗结构将指令和数据分离，但ddr芯片只有一块，所以如果要用的话，可以分开两部分统一编制，一部分存数据，一部分存指令，然后两块BRAM做ddr中数据和指令的高速缓存。

### IO接口

IO接口都需要挂载到数据总线上（通过连接到数据总线译码器），由数据总线译码器使能选通。

#### 外设接口

负责屏蔽cpu和外设的速度、协议差异。

主要包括

USB-HID接口，按钮接口，滑动开关接口（需要防抖动），LED接口，七段数码管接口，用户按键接口等等

#### 中断控制器

遵循RISC-V的PLIC中断控制器规范，是一种内存映射IO设备。

cpu通过内存地址访问其寄存器。

各外设的中断信号直接发往中断控制器，由中断控制器进行总裁并发往cpu。

nexys板载设备比较少，中断源定义为16个足够了

## 接口契约

本节定义系统顶层各组件之间的端口规范。所有总线事务采用请求/应答握手机制。实现方必须遵循本契约中定义的信号名称、位宽、方向和时序约定。

### 命名与方向约定

- 指令总线信号统一以 `imem_` 为前缀，数据总线信号统一以 `dmem_` 为前缀。
- 同一信号线在所有组件上使用**相同的端口名**——不存在一侧叫 `dmem_we`、另一侧叫 `we` 的情况。
- 控制信号均为高有效，除非名称中显式含 `_n`后缀（低有效）。
- 方向标注以**目标组件**为视角：`out`=该组件驱动，`in`=该组件接收。
- 复位默认值：各输出信号在 `rst_n` 有效期间必须保持其组件声明复位值；未声明的默认为 `0`。

### 握手机制

所有总线事务统一采用 req/ack 同步握手：

1. **发起方**在时钟上升沿驱动 `_req` 及全部关联信号（地址、数据、控制位），并在 `_req` 有效期间保持所有信号稳定。
2. **响应方**在操作完成时在时钟上升沿置 `_ack=1`；若未完成则保持 `_ack=0`，发起方应等待。
3. 事务于 `_req==1` 且 `_ack==1` 的时钟上升沿完成，发起方可在下一周期撤销 `_req`。
4. 若需返回数据（读），响应方**与 `_ack` 同周期**驱动数据。

**本契约不假设任何"同周期必完成"的时序要求**——响应方可插入任意数量的等待周期。BRAM 等组合逻辑实现可自然地单周期完成，但这属于实现特性而非契约约束。

### 时钟与复位

| 端口      | 位宽 | 方向 | 规范                                               |
| --------- | :--: | :--: | -------------------------------------------------- |
| `clk`   |  1  |  in  | 全局时钟，所有时序逻辑在上升沿采样                 |
| `rst_n` |  1  |  in  | 低有效全局复位。复位释放后第 1 个上升沿起 CPU 启动 |

约束：

- 系统内所有组件接入同一 `clk` 和 `rst_n`，单一同步时钟域，不引入门控时钟。
- 复位最小持续时间 ≥ 16 个 `clk` 周期。
- 复位期间所有输出信号保持各自复位默认值。

### 指令总线

CPU 通过点对点连接从指令存储器取指。

| 端口          | 位宽 |   方向   | 规范                               |
| ------------- | :--: | :-------: | ---------------------------------- |
| `imem_req`  |  1  | CPU→IMEM | 取指请求，高有效                   |
| `imem_addr` |  32  | CPU→IMEM | 取指字节地址，必须按字对齐         |
| `imem_ack`  |  1  | IMEM→CPU | 指令就绪，本周期`imem_data` 有效 |
| `imem_data` |  32  | IMEM→CPU | 指令字                             |

握手：

- CPU 驱动 `imem_addr` 并置 `imem_req=1` 发起取指。`imem_req` 和 `imem_addr` 保持稳定直至 `imem_ack` 置位。
- 指令存储器在指令数据就绪时置 `imem_ack=1` 并驱动 `imem_data`。
- 指令存储器可通过保持 `imem_ack=0` 插入任意等待周期。
- 复位后 `imem_ack=0`，`imem_data=0`。

### 数据总线

CPU 通过数据总线译码器访问数据存储器及全部内存映射 IO 设备。CPU 为主端，译码器为从端。

| 端口           | 位宽 |   方向   | 规范                                                       |
| -------------- | :--: | :-------: | ---------------------------------------------------------- |
| `dmem_req`   |  1  | CPU→总线 | 访存请求，高有效                                           |
| `dmem_addr`  |  32  | CPU→总线 | 访存字节地址                                               |
| `dmem_we`    |  1  | CPU→总线 | 写使能，高=写、低=读                                       |
| `dmem_wdata` |  32  | CPU→总线 | 写入数据（写操作时有效）                                   |
| `dmem_width` |  2  | CPU→总线 | 访存宽度：`00`=字，`01`=半字，`10`=字节，`11`=保留 |
| `dmem_ack`   |  1  | 总线→CPU | 访存完成，本周期`dmem_rdata` 有效                        |
| `dmem_rdata` |  32  | 总线→CPU | 读出数据                                                   |
| `dmem_fault` |  1  | 总线→CPU | 访存错误（与`dmem_ack` 同时置位有效）                    |

握手：

- CPU 驱动 `dmem_req=1` 连同 `dmem_addr`、`dmem_we`、`dmem_wdata`（写时）、`dmem_width`，发起一次访存事务。
- CPU 保持全部输出信号稳定直至总线返回 `dmem_ack=1`。
- **读事务**：总线在数据就绪时置 `dmem_ack=1` 并驱动 `dmem_rdata`。
- **写事务**：总线在写入完成时置 `dmem_ack=1`。
- 总线可通过保持 `dmem_ack=0` 插入任意等待周期。
- 若地址未命中任何设备或发生访问违例，总线在置 `dmem_ack=1` 时同步置 `dmem_fault=1`；此时 `dmem_rdata` 无定义。

`dmem_rdata` 始终返回 32 位原始数据——不进行符号扩展。字节或半字读取时，目标数据出现在地址对应的字节通道（`addr[1:0]==2'b01` 时字节在 `rdata[15:8]`），由 CPU 根据当前指令自行提取并完成符号/零扩展。

### 总线译码器与地址映射

译码器根据 `dmem_addr` 高位译码，为每个事务产生唯一条有效的片选线 `dmem_cs[n:0]`。未命中任何区间视为无设备，译码器返回 `dmem_ack=1` + `dmem_fault=1`。

**译码器工作流程**：

1. 接收 CPU 的 `dmem_req`、`dmem_addr`、`dmem_we`、`dmem_wdata`、`dmem_width`。
2. 根据 `dmem_addr` 高位译码，置位对应设备的 `dmem_cs`，向所有设备转发 `dmem_addr`、`dmem_we`、`dmem_wdata`、`dmem_width`。
3. 等待被选中设备的 `dmem_ack` 响应。
4. 将被选中设备的 `dmem_ack`、`dmem_rdata`、`dmem_fault` 转发给 CPU 的数据总线。

地址映射表：

| 设备        | 基址            | 结束地址        |  大小  | `dmem_cs` |
| ----------- | --------------- | --------------- | :-----: | :---------: |
| 数据存储器  | `0x0000_0000` | `0x07FF_FFFF` | 128 MiB |  `cs[0]`  |
| PS/2 控制器 | `0x8000_0000` | `0x8000_0007` |   8 B   |  `cs[1]`  |
| 滑动开关    | `0x8000_0008` | `0x8000_000B` |   4 B   |  `cs[2]`  |
| LED         | `0x8000_000C` | `0x8000_000F` |   4 B   |  `cs[3]`  |
| 数码管      | `0x8000_0010` | `0x8000_002F` |  32 B  |  `cs[4]`  |
| 用户按键    | `0x8000_0030` | `0x8000_0033` |   4 B   |  `cs[5]`  |
| 中断控制器  | `0x8100_0000` | `0x8120_0007` | ~2 MiB |  `cs[6]`  |

### 设备接口统一规范

译码器与各设备接口之间的信号命名与数据总线**完全一致**。所有内存映射的设备接口均实现以下端口集：

| 端口           | 位宽 | 方向 | 规范                                                    |
| -------------- | :--: | :--: | ------------------------------------------------------- |
| `clk`        |  1  |  in  | 全局时钟                                                |
| `rst_n`      |  1  |  in  | 全局复位                                                |
| `dmem_cs`    |  1  |  in  | 片选，高有效——仅当`dmem_cs==1` 时设备响应           |
| `dmem_addr`  |  32  |  in  | 访存字节地址（设备仅关注与自身地址窗口对应的低位偏移）  |
| `dmem_we`    |  1  |  in  | 写使能，高=写、低=读                                    |
| `dmem_wdata` |  32  |  in  | 写入的数据                                              |
| `dmem_width` |  2  |  in  | 访存宽度，同数据总线定义                                |
| `dmem_ack`   |  1  | out | 操作完成，读操作时本周期`dmem_rdata` 有效             |
| `dmem_rdata` |  32  | out | 读出的数据                                              |
| `dmem_fault` |  1  | out | 访问错误（无效偏移、写只读寄存器等）                    |
| `dmem_irq`   |  1  | out | 中断请求线，直连 PLIC 对应中断源 ID；无中断需求则恒为 0 |

约束：

- `dmem_cs==0` 时设备忽略全部输入，输出 `dmem_ack=0`、`dmem_rdata=0`、`dmem_fault=0`。
- `dmem_cs==1` 时设备可插入等待周期（保持 `dmem_ack=0`），完成后置 `dmem_ack=1`。
- 译码器汇总`dmem_ack`、`dmem_rdata`、`dmem_fault` 并转发到 CPU 端数据总线。
- 复位默认：`dmem_ack=0`，`dmem_rdata=0`，`dmem_fault=0`，`dmem_irq=0`。

#### 数据存储器

地址范围 `0x0000_0000`—`0x07FF_FFFF`（128 MiB 地址空间，实际物理容量由实现方决定）。

- **读**：根据 `dmem_addr` 和 `dmem_width` 返回对齐字内的对应数据。
- **写**：根据 `dmem_addr` 和 `dmem_width` 进行字节/半字/字写入。
- **fault**：地址超出实际物理容量时置 `dmem_fault=1`。
- **irq**：恒为 0，无中断。
- BRAM 实现可单周期完成（`dmem_ack` 在 `dmem_cs` 同周期返回）；DDR 实现将插入等待周期。无论何种实现，契约仅要求 `dmem_ack` 在操作完成后置位。

#### PS/2 控制器

地址范围 `0x8000_0000`—`0x8000_0007`（8 字节，2 个 32-bit 寄存器）。设备使用的地址位：`dmem_addr[2]` 区分寄存器。

寄存器：

|  偏移  | 名称          | 访问 | 说明                                                                                                    |
| :-----: | ------------- | :--: | ------------------------------------------------------------------------------------------------------- |
| `0x0` | `PS2_CTRL`  | R/W | bit 0：接口使能；bit 8：接收中断使能。bits[31:9,7:1] 保留，读返回 0。                                   |
| `0x4` | `PS2_RDATA` |  R  | bit[7:0]：接收字节；bit 8：数据有效（读到 1 表示新数据，读后硬件自动清零）。bits[31:9] 保留，读返回 0。 |

- `dmem_fault`：写 `PS2_RDATA` 或访问偏移 `0x8` 及以上时置位。
- `dmem_irq`：新数据到达 且 接收中断使能（`PS2_CTRL[8]==1`）时置位，直连 PLIC 源 ID 2。
- 复位默认：`PS2_CTRL=0`，`PS2_RDATA=0`。

#### 滑动开关接口

地址范围 `0x8000_0008`—`0x8000_000B`（4 字节，单寄存器，只读）。

| 位域        | 说明                            |
| ----------- | ------------------------------- |
| `[15:0]`  | SW[15:0] 当前开关状态（1=导通） |
| `[31:16]` | 保留，读返回 0                  |

- **读**：返回当前开关状态（消抖后的稳定值）。
- **写**：`dmem_fault=1`。
- `dmem_irq`：任意开关状态改变（消抖后）时置位，直连 PLIC 源 ID 10。电平保持直至通过 Claim/Complete 清除。

#### LED 接口

地址范围 `0x8000_000C`—`0x8000_000F`（4 字节，单寄存器，可读写）。

| 位域        | 说明                   |
| ----------- | ---------------------- |
| `[15:0]`  | LED[15:0] 输出（1=亮） |
| `[31:16]` | 保留，写忽略、读返回 0 |

- 复位默认 `0x0000_0000`（全灭）。
- `dmem_irq`：恒为 0。

#### 数码管接口

地址范围 `0x8000_0010`—`0x8000_002F`（32 字节，8 个 32-bit 寄存器）。设备使用的地址位：`dmem_addr[4:2]`（3 位索引，对应 AN[0]—AN[7]）。

寄存器：

|   偏移   | 名称         | 访问 | 说明                                                           |
| :------: | ------------ | :--: | -------------------------------------------------------------- |
| `0x00` | `SEG_CTRL` | R/W | bit 0：全局使能（1=显示，0=全灭）。bits[31:1] 保留，读返回 0。 |
| `0x04` | `SEG_DIG0` | R/W | AN[0] 段码                                                     |
| `0x08` | `SEG_DIG1` | R/W | AN[1] 段码                                                     |
| `0x0C` | `SEG_DIG2` | R/W | AN[2] 段码                                                     |
| `0x10` | `SEG_DIG3` | R/W | AN[3] 段码                                                     |
| `0x14` | `SEG_DIG4` | R/W | AN[4] 段码                                                     |
| `0x18` | `SEG_DIG5` | R/W | AN[5] 段码                                                     |
| `0x1C` | `SEG_DIG6` | R/W | AN[6] 段码                                                     |
| `0x20` | `SEG_DIG7` | R/W | AN[7] 段码                                                     |

段码位分配（`SEG_DIGx`）：bit[6:0] 为 A—G 段选，**低有效**（0=亮，1=灭）；bit 7 为小数点，低有效。

| bit 6 | bit 5 | bit 4 | bit 3 | bit 2 | bit 1 | bit 0 |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: |
|   A   |   B   |   C   |   D   |   E   |   F   |   G   |

- 复位默认：`SEG_CTRL=0x0000_0000`，`SEG_DIGx=0x0000_00FF`（全灭）。
- `SEG_CTRL[0]==0` 时强制全灭，寄存器值不变。
- `dmem_fault`：访问偏移 `0x24`—`0x2C`（未定义寄存器）时置位。
- `dmem_irq`：恒为 0。

#### 用户按键接口

地址范围 `0x8000_0030`—`0x8000_0033`（4 字节，单寄存器，只读）。

| 位域       | 说明                  |
| ---------- | --------------------- |
| `[0]`    | BTN_L（左），按下为 1 |
| `[1]`    | BTN_D（下），按下为 1 |
| `[2]`    | BTN_R（右），按下为 1 |
| `[3]`    | BTN_U（上），按下为 1 |
| `[4]`    | BTN_C（中），按下为 1 |
| `[31:5]` | 保留，读返回 0        |

- **读**：返回消抖后的当前按键状态。
- **写**：`dmem_fault=1`。
- `dmem_irq`：任意按键按下（消抖后）时置位，直连 PLIC 源 ID 8。电平保持直至通过 Claim/Complete 清除。

### 中断系统

#### 中断源分配

| 中断 ID | 外设来源               | 说明                    | 建议优先级 |
| :-----: | ---------------------- | ----------------------- | :--------: |
|    0    | （保留）               | Claim 返回 0 表示无中断 |     —     |
|    1    | UART                   | 串口收发                |     7     |
|    2    | PS/2（键盘/鼠标）      | 人机交互输入            |     6     |
|    3    | 定时器 0（系统心跳）   | 操作系统节拍            |     5     |
|    4    | 定时器 1（通用）       | 用户自定义定时          |     4     |
|    5    | 以太网                 | 网络数据收发            |     5     |
|    6    | 加速度计               | 运动检测                |     3     |
|    7    | 温度传感器             | 温度阈值                |     2     |
|    8    | GPIO（按键/外部引脚）  | 按键中断                |     4     |
|    9    | MicroSD 卡             | 数据传输完成            |     3     |
|   10   | 16路滑动开关           | 开关状态改变            |     6     |
|   11   | QSPI Flash（用户模式） | 用户 QSPI 访问完成      |     2     |
|   12   | PDM 麦克风             | 音频缓冲区满            |     1     |

未实现的中断源，其 `dmem_irq`/`irq_*` 线恒为 0。

#### 中断请求线

各外设的中断请求线直连 PLIC 的对应源 ID 输入。未实现的源输入端接地（恒 0）。

| PLIC 源 ID | 信号来源                    | 信号说明         |
| :--------: | --------------------------- | ---------------- |
|     2     | PS/2 控制器`dmem_irq`     | 接收数据就绪     |
|     3     | CPU 定时器 0`irq_timer0`  | 系统心跳比较匹配 |
|     4     | CPU 定时器 1`irq_timer1`  | 通用定时比较匹配 |
|     8     | 用户按键接口`dmem_irq`    | 有按键按下       |
|     10     | 滑动开关接口`dmem_irq`    | 有开关状态改变   |
|     15     | CPU CSR 软件中断`irq_swi` | 软件触发         |

每条 `irq` 为电平信号（高有效），保持拉高直至被软件通过 PLIC Claim/Complete 流程服务完毕后置低。

#### PLIC（中断控制器）寄存器映射

PLIC 内存映射于基址 `0x8100_0000`。所有寄存器均为 32-bit、字对齐。

|   基址偏移   | 寄存器             | 访问 | 说明                                                                              |
| :----------: | ------------------ | :--: | --------------------------------------------------------------------------------- |
| `0x000004` | `PRIORITY[1]`    | R/W | 中断源 1 优先级（0=禁用该源）                                                     |
| `0x000008` | `PRIORITY[2]`    | R/W | 中断源 2 优先级                                                                   |
|      …      | …                 |      | 每源 4 字节，偏移=4×ID                                                           |
| `0x00003C` | `PRIORITY[15]`   | R/W | 中断源 15 优先级                                                                  |
| `0x001000` | `PENDING`        |  R  | 中断待决位（bit N=源 N 待决）                                                     |
| `0x002000` | `ENABLE`         | R/W | Hart 0 使能位（bit N=源 N 使能）                                                  |
| `0x200000` | `THRESHOLD`      | R/W | 优先级阈值（0=允许所有）                                                          |
| `0x200004` | `CLAIM/COMPLETE` | R/W | 读：返回最高优先级的待决中断 ID（无则 0）；<br />写：写入已完成服务的 ID 释放中断 |

- 所有未定义偏移位置读返回 0、写忽略。
- `PENDING` 为硬件控制，写入忽略。
- `CLAIM` 只读、`COMPLETE` 只写（同一地址分读写语义）。

#### 定时器与软件中断

定时器 0 / 定时器 1 的控制寄存器（`mtime`、`mtimecmp`）和软件中断触发均为 CPU 内部逻辑（CSR 或内部寄存器实现）。其 `irq_timer0`、`irq_timer1`、`irq_swi` 分别接至 PLIC 源 3、4、15。

同时，CPU 的机器态中断输入线 `mtip`（定时器中断）和 `msip`（软件中断）为独立于 PLIC 的直连通路，遵循 RISC-V 特权架构标准。两者与 PLIC 路由的中断线并存——`mtip`/`msip` 提供低延迟路径，PLIC 路由提供统一仲裁。

#### PLIC/Core 中断信号 → CPU

| 端口     | 位宽 | 方向        | 规范               |
| -------- | :--: | ----------- | ------------------ |
| `meip` |  1  | PLIC → CPU | 机器态外部中断待决 |
| `mtip` |  1  | 内部 → CPU | 机器态定时器中断   |
| `msip` |  1  | 内部 → CPU | 机器态软件中断     |

## 生命周期

### BootLoader

随 bitStream 预先写到iRAM 上的固件。

CPU 的 PC 初值为 BootLoader 的地址，这样系统一启动，bootloader 就会运行。

目前不用做真正的boot，因为用户代码需要从其他地方读取比较麻烦。

可以直接把测试代码写到iram0x0处cpu上电自动执行测试程序。
